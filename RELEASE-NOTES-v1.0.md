# ARVANN v1.0 — Release Notes

**Tagged:** commit `f1a5fed`
**Platforms:** Android release APK (89 MB), iOS Xcode archive pending

## What's in v1.0

ARVANN's first shippable release. End-to-end auth (email + Apple Sign-In), profile, inventory with product variants, company management with verification, chat, support, ads, and admin Ops Console — all behind a working backend at `api.arvann.in`.

## Final sprint fixes (last 24h before tagging)

- **Apple Sign-In → save-phone gate** unstuck. Duplicate route registration in AppNavigator was preventing the gate screen from unmounting after a successful save. Added `key={authSection}` on the navigator + optimistic phone injection + deterministic gate-flag clear in `refreshUser()`.
- **Profile editor safe-area** fixed. `react-native-safe-area-context`'s `<SafeAreaView>` doesn't read insets inside a React Native `<Modal>` — switched to `useSafeAreaInsets()` hook with manual padding.
- **Variant form sheet** now renders the form fields. Sheet had `maxHeight: 88%` with no fixed height — inner ScrollView's `flex: 1` collapsed to zero. Switched to explicit `height: 85%`.
- **"Create first variant" button** now reacts. Moved empty state out of FlatList's `ListEmptyComponent` (which can intercept taps on iOS) and rendered it as a normal ScrollView child.
- **Logo / avatar upload** ("Base64 content is required" error) fixed. Frontend was sending multipart FormData; backend validator expected JSON with base64 `content` field. Frontend now reads the file as base64 via `expo-file-system/legacy` and POSTs JSON.
- **Profile phone field locked.** OTP-verified phone change attempted, reverted to display-only after a deploy mismatch. To change phone, contact support or use the dedicated `AddMobileNumberScreen` social-signup gate.
- **Verification screen gate** for normal accounts. Opens a popup ("Only trader or manufacturer companies can request verification") instead of an empty form.

## Build artifacts

- iOS: not yet archived. Run Xcode → Product → Archive → upload via App Store Connect to ship the v1.0 build to TestFlight.

## Known issues deferred to v1.1+

- `signInWithApple` derives the phone-collection gate from the Apple response rather than reconciling with server truth.
- `handleBusinessSubmit` sets user state from the signup-complete response without a refresh roundtrip.
- `httpClient` silently swallows failed `response.json()` parses into `undefined`.
- `ACTIVE_COMPANY_REQUIRED` auto-retry doesn't refresh `AuthProvider`'s user state after restoring the active company.
- Pre-existing log warnings: `LinearGradient colors and locations props should be arrays of the same length`; `Encountered two children with the same key 69bd18cc227bb0252c0495fb` in some list render.
